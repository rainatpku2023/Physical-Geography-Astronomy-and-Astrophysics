### 0 《地图学与GIS基础》前言
+ 地理学：地理学是研究地球表面的地理环境中各种自然现象和人文现象，以及它们之间相互关系的学科
+ 《地图学与GIS基础》课程重要性（E. L. Ullman）
    - 地图学是地理学的第二语言：以图解形式表达地学研究成果
    - 地理信息系统是地理学的第三语言：是地图学的延伸，用于信息技术存储、分析、显示地理信息
+ 学科性质：交叉性，实践性，有前景

### 1 第一章 绪论
#### 1.1 地图学与地理信息系统的基本概念
+ 地图学的三大法则：数学法则（地图投影）、制图综合法则（制图综合）、符号法则（地图语言/符号）
+ 地图学与地理信息系统的历史
    - 巴比伦地图（前900-前600）：最早的世界地图
    - 托勒密的世界地图（约90-168）
    - 《禹贡地域图》（224-281）：晋代地图学家、中国古代地图的绘制理论创始人裴秀的作品
        * 计里画方：将地表面视为平面、不考虑地表曲率，也就是方里网
        * **<font style="color:#DF2A3F;">制图六体：分率（比例尺），准望（方位），道里（距离），高下（地势起伏），方邪（倾斜角度），迂直（河流、道路的曲直）</font>**
    - 《禹迹图》（南宋）：南宋石刻
    - 墨卡托的世界地图（1538）
    - 霍乱地图（1854）：斯诺医生终止霍乱
+ 地理信息系统的基本概念
    - 陈述彭院士：地理信息系统可简单定义为用于采集、模拟、处理、检索、分析和表达地理空间数据的计算机系统。
    - Goodchild：获取、存储、管理、分析和现实有关地理空间数据的计算机系统。
    - Tomlinson：从现实世界中采集、存储、提取、转换和显示空间数据的一组有力的工具。
+ 对于地理信息系统的三种不同观点
    - 数据库视角（GeoDatabase）：计算机学派，GIS作为工具存储和组织空间数据
    - 地图视角（GeoVisualization）：制图学派，GIS作为工具将空间数据展示为地图
    - 空间分析观（Spatial Analysis）：地理学派，GIS作为工具分析和诠释空间数据
+ 地理信息系统的组成（6个部分）：计算机硬件系统、计算机软件系统、地理空间数据、系统开发管理和使用人员、应用分析模型和网络

#### 1.2 地理信息系统功能与应用
+ 数据采集与编辑
    - 矢量数据（Vector Data）编辑，栅格数据（Rastor/Grid/Image Data）编辑，属性数据（Attribute Data）
+ 数据组织与存储：空间、属性数据一般分开存储，然后用标识码连接
    - 空间数据组织方式：矢量模型，栅格模型，栅格/矢量混合模型
    - 属性数据组织方式：关系数据结构，层次结构，网状结构
+ 数据处理与变换：数据格式化、数据转换、数据概括
+ 空间分析和统计：地理信息系统的核心功能，也是地理信息系统与其他计算机系统的根本区别
+ 地图制作与显示

#### 1.3 地图学与GIS发展历史
+ 四个阶段：创新阶段、商业化阶段、开放普及阶段、社会化阶段

    - 1963：Roger Tomlinson和同事共同开发CGIS，加拿大地理信息系统

    - 2005：Google Earth发布
    - 2010：ESRI发布云GIS服务平台

#### 1.4 地理信息系统在学科中的地位
+ 学科门类：理学/工学
    - 理学 -> 地理学 ->地图学与地理信息系统
    - 工学 -> 测绘科学与技术 -> 地图制图学与地理信息工程

### 2 第二章 地球与地球坐标系
#### 2.0 空间参考的重要性
+ 空间参考（Spatial Reference）描述了一个地物在地球上的真实位置，而位置依赖于既定的坐标系来表示
+ 坐标系（CS）包括地理坐标系（GCS）和投影坐标系（PCS）
    - GCS包括天文地理坐标、大地地理坐标、空间直角坐标系

#### 2.1 地球的形状与大小
+ 地球自然表面（包围地球体）
+ 大地水准面（包围大地球体）：假定海水处于完全静止状态改下，把它延伸到大陆内部，形成包围整个地球的连续闭合表面，它处处与铅垂线垂直。
+ 地球椭球体：使用旋转椭球体代替大地球体
    - 参考椭球体：和大地水准面符合最好的一个地球椭球退
    - 椭球定位：将选定椭球体摆到与大地水准面最贴近的位置上——即对地球形体进行了逼近
    - 椭球定向：确定椭球旋转轴的方向
    - 椭球定位和定向分为区域性（参心）定位和全球（地心）定位 -> 地心定位要求在全球范围内椭球面与大地水准面最佳吻合，同时要求椭球中心与地球质心一致。
+ 圆球体：制作小比例尺地图时（小于1:500万），即制图尺度接近地球大小，忽略地球扁率；比例尺大于1:100万时就必须将地球视为椭球体了
+ 各种参数的计算：扁率、第一偏心率、第二偏心率

#### 2.2 地球与地球坐标系
+ 国家大地控制网：GPS控制网、水平大地控制网、高程控制网
+ 地理坐标系
    - **<font style="color:#DF2A3F;">天文地理坐标：以大地水准面为参考面，以铅垂线为依据，用天文经度和天文纬度表示位置</font>**
    - **<font style="color:#DF2A3F;">大地地理坐标：以椭球体面为参考面，以法线为依据，用大地经度和大地纬度表示位置</font>**
    - 大地空间直角坐标系
+ 常见的地理坐标系
    - 1954年北京坐标系：参心坐标系，多点定位法，大地原点普尔科沃，高程用1954年青岛验潮站求出的黄海平均海水面，传统大地测量技术，二维
    - 1980年国家大地坐标系：参心坐标系，高程异常值平方和最小原则求解参数，大地原点陕西省泾阳县永乐镇，高程用1956年青岛验潮站求出的黄海平均水面为基准，传统大地测量技术，二维
    - CGCS2000年国家大地坐标系：地心坐标系，参考历元2000.0，三维
    - WGS84坐标系：地心坐标系，国际采用，为GPS全球定位系统使用而建立，三维
+ 北京54和西安80的问题
    - 坐标维的矛盾：只提供二维坐标，不适用三维定位的需求
    - 精度矛盾：西安80定位精度低
    - 坐标轴的矛盾：短半轴与国际地球参考系统（ITRS）或与WGS84的指向不同
+ 高程系统
    - 正高系统：以大地水准面为基准面，铅垂线方向
    - 正常高系统：以似大地水准面为基准面，正常重力线方向
    - 大地高程系统（大地高）：以参考椭球面为基准面，法线方向
+ 其他组成元素
    - 水准原点：与验潮站相互联系的高程起算点
    - 国家高程基准：是推算国家统一高程控制网中所有的水准高程的起算依据，包括一个国家水准基面和一个永久性的水准原点（我国的在银海大世界）
+ 各种误差
    - 垂线偏差：铅垂线和法线不一致，产生偏角，称为垂线偏差
    - 大地水准面差距：大地高减去正高（若为正说明参考椭球面在大地水准面下）
    - 高程异常：大地高减去正常高（若为正说明参考椭球面在似大地水准面下）
+ 我国的高程系
    - 1956年黄海高程系：1950-1956年7年验潮站资料推算出来的黄海海水面的平均值，作为我国高程的基准面。青岛观象山上的国家水准原点，确定其高程为72.289米。
    - 1985年国家高程基准：1953-1979年共27年的潮汐观测资料求得的平均值，推算出水准原点的高程为72.260米，可认为海水上涨了28.6毫米。

#### 附录：中华人民共和国国界标准画法
+ 南海诸岛作为附图表示或者整体表示
+ 一般来说：要区分未定国界、必须表示南海诸岛、必须画出南海诸岛断续线、必须表示钓鱼岛和赤尾屿
+ 轮廓线和填充色表示中国版图：未定国界和南海诸岛断续线可不表示，但是必须表示南海诸岛、钓鱼岛和赤尾屿；注意使用相同颜色
+ 中国海域界线标准画法
    - 比例尺大于1:1亿（即比较精细）：9（南海诸岛断续线）+1（中日海上界线）=10段，不能漏画钓鱼岛和赤尾屿
    - 比例尺小于1:1亿（即比较粗略）：7段（逆时针删去2和7）+1（中日海上界线）=8段，钓鱼岛和赤尾屿表示不强求

### 3 第三章 地球投影与坐标转换
#### 3.0 地图投影的基本问题
+ 地图投影的定义：按照一定的数学法则，将地球椭球面的经纬网转换到平面上建立地球表面上地理坐标与地图上对应的平面直角坐标之间一一对应的函数关系
+ 地图投影研究的主要内容：投影变换、变形规律、投影转换、投影选择
+ 投影变形的描述
    - 解析描述
        * 长度比与长度变形：最大长度比a和最小长度比b（两者称为主方向长度比），经线长度比m和纬线长度比n，变形椭圆方位角α；计算时投影后的结果放在分子上，原来放在分母上
        * 面积比与面积变形：投影后面积减去原来面积再减去1，结果为正（正变形）说明面积增大，否则（负变形）说明面积减小
        * 角度变形：投影后角度减去原来角度，一般只研究角度变形最大值ω
    - 图解描述
        * 变形椭圆
        * 等变形线

#### 3.1 地图投影的分类和选择
+ 变形性质分类
    - 等角投影：用于航海、洋流和风向图等；交通图，航海图，航空图，军用等
    - 等积投影：用于对面积要求较高的自然社会经济地图；区划图，类型图等
    - 任意投影：折中；中小学的教学用图
+ 地图投影的分类
    - 按照数学法则，可以分为几何投影和非几何投影（重点在于光线会不会拐弯）
    - 几何投影的分类标准
        * 光源位置与接收面的形状：圆柱投影（cylindrical）、圆锥投影（conic）、方位投影（azimuthal）
        * 接收面的旋转轴的与地球旋转轴的位置关系：正轴（重合）、横轴（垂直）、斜轴（斜交）
        * 接收面与参考椭球面的位置关系：切投影、割投影
    - 地图投影特点的描述内容
        * 经线在地图上的形状，距离比例 -> 注意区分成比例与相等的区别
        * 纬线在地图上的形状，距离比例
        * 其他描述
+ 常见选择：等边兴县尽量与制图区域形状一致
    - 极地位置：正轴方位投影
    - 赤道附近：横轴方位投影或者正轴圆柱投影
    - 半球地图：横轴方位投影
    - 中纬地区：正轴圆锥投影或者斜轴方位投影
    - 低纬度地区：正轴圆柱投影
+ 特殊要求和约定俗成的投影
    - 世界时区图：正轴等角圆柱投影
    - 海洋地图、宇航地图：等角圆柱投影
    - 航空基地图：等距离方位投影（指将航空基地放在中心的情况）
    - 地形图：等角横切/割圆柱投影

#### 3.2 墨卡托投影
+ 定义：正轴等角圆柱投影，并没有规定是切还是割
    - 等角条件：经向和纬向变形率相同（m=n）
+ 应用：大圆弧线是球面上两点间的最短距离
+ 衍生：Web Mercator（900913），本质上是一种对墨卡托投影的改良
    - 基准面为椭球
    - 投影参数选用正球体
    - 绘图范围为正方向
    - 纬度范围南北纬85.06°之间
+ 各种奇奇怪怪的投影：巴克敏斯特·富勒地图投影，全球离散网格DDGS，Uber的H3空间索引

#### 3.3 高斯克吕格投影和UTM投影（看图！！！）
+ 高斯克吕格投影：分为6度和3度两种，第一个带中心线都是东经3度，都是从西向东走一圈
    - 性质：中央经线与赤道投影相互垂直，等角性质，中央经线长度不变
    - 距离变形规律：中央经线不变形，**纬度越高变形越小**，经度与中央经线差距越大变形越大
    - 我国国家地形图11中比例尺（**1:500,1:1000,1:2000,1:5000,1:10000**,1:25000,1:50000,1:100000,1:250000,1:500000,1:1000000）
        * 加粗部分用3度分带，没加粗部分用6度分带
        * 奇怪的是1:2000这个比例尺
    - 衍生：高斯平面直角坐标系，纵轴为x横轴为y非常奇怪，投影带西移500km
    - 次衍生：方里网，根据坐标纵线和坐标横线按照整公里绘出；相邻方里网重叠的必要性
    - 三北指针：真北方向（地理北），磁北方向，坐标北方向
        * 夹角：磁偏角、子午线收敛角、磁坐偏角
+ UTM投影：只有6度，第一个带中心都是西经177度，从180度经线开始，从西向东走一圈
    - 割圆柱投影，两条标准线，改善了低纬度和中纬度变形问题

#### 3.4 兰伯特正轴等角圆锥投影（看图！！！）
+ 1:100万地形图采用双标准纬线正轴等角割圆柱投影
    - 双标准纬线：没有变形的纬线总共有2条
+ 1:100万地形图分幅问题：一般来说经度差为6度，纬度差为4度；还是从180度经线从动详细计数
    - **<font style="color:#DF2A3F;">60-76度2幅横向合并</font>**
    - **<font style="color:#DF2A3F;">76-88度4幅横向合并</font>**
    - **<font style="color:#DF2A3F;">88度以上整体为1幅</font>**

#### 3.5 空间坐标转换
+ 四参数转换：一个高斯平面直角坐标转换为另外一个**<font style="color:#DF2A3F;">高斯平面直角坐标</font>** -> 面向平面
    - 因此有2个平移参数，1个旋转参数，1个尺寸参数
+ 七参数转换模型：一个空间直角坐标转换为另外一个坐标系的**<font style="color:#DF2A3F;">空间直角坐标 </font>**-> 面向空间
    - 因此有3个平移参数，3个旋转参数，1个尺寸参数
+ 三参数转换模型：七参数的弱化版，不考虑旋转参数和尺寸参数；获取较为容易两个点即可
+ 数值法转换：空间校正或地图配准

#### 附录：各种等高线
+ 首曲线（基本等高线）：最基础，根据平均海水面起算绘制而成，棕色细实线
+ 计曲线（加粗等高线）：每隔4条画一个，方便计数，棕色粗直线
+ 间曲线（半距等高线）：1/2等高距插出来，棕色长虚线
+ 助曲线（辅助等高线）：1/4等高距插出来，棕色短虚线

### 4 第四章 空间数据模型与数据结构
#### 4.0 基本概念
+ 数据：指某一目标定性、定量描述的原始资料，包括数字、文字、符号、图形、图像以及它们能转换成的数据等形式
+ 信息：数据、消息包含的意义

#### 4.1 空间数据特征（空间数据的基本性质）
+ 基本描述：以空间位置为参照（空间相关性/在特定的坐标和空间参照系下讨论），尺度，动态性
+ 面向类型：空间特征（定位数据），属性特征（非定位数据），时间特征
    - 专题属性：定名量表（nominal），顺序量表（ordinal），间隔量表（interval），比率量表（ratio）
    - 比率量表常常使用等比数列

#### 4.2 空间数据模型
+ 场模型、要素模型、网络模型
    - 网络模型：描述离散的对象，通过目标之间相关连接（如路径）相互连接多个空间实体，可以看成是基于点对象和线对象及其拓扑关系的集合的描述
+ 其他扩展模型：时空数据模型与三维数据模型
+ 场模型的困难：离散化而导致的误差，不支持空（但支持NODATA）
    - 离散方式：离散点、等高线、空间镶嵌（栅格、TIN、Voronoi，后两者互为互补图）
+ 网络模型：参照《数据结构与算法》笔记
+ 场模型对应栅格数据结构，要素模型对应矢量数据结构
    - 栅格数据模型的问题：像元、像素、像元的值（混合像元问题）、像元的大小（分辨率问题）、位置；三维栅格数据模型称为体素（voxel）
+ 时空数据模型：三种变化（属性变化，空间特性即位置变化，两者同时变化）

#### 4.4 空间数据结构
+ 空间数据结构：适合于计算机系统存储、管理和处理的地理数据的逻辑结构，是地理实体的空间排列和相互关系的抽象描述
+ 栅格数据结构的编码：直接栅格编码，游程长度编码，四叉树编码（根节点是整个区域，叶节点是不能在分割的块，深度是分几级）
    - **<font style="color:#DF2A3F;">基于Morton码线性四叉树编码：得到Morton码后先转化为二进制，其中奇数位用列号填充，偶数位用行号填充。</font>**
+ **<font style="color:#DF2A3F;">（多边形）矢量数据结构的编码：坐标序列法（一个一个按顺序列出），树状索引编码，拓扑结构编码（点与线的关系）</font>**
+ 空间关系
    - 拓扑空间关系：实体之件相邻、相通、包含和相交的关系
        * 邻接关系：高维实体共用部分低维成分，面向同类元素
        * 关联关系：面向不同类元素之间的关系
        * 包含关系：同类但不同级别的元素的大小关系。
    - 顺序空间关系：地理空间排列关系（前后上下左右东南西北等）
    - 度量空间关系：多远多近

### 6 空间数据管理
#### 6.0 空间数据组织（和数据结构要相互区分）
+ 图层（Layer）：GIS系统中基本的用来组织和存储空间数据的单位
+ 文件（File）：优势在于灵活，缺点是需要用到各种应用程序进行处理，查询较为复杂，共同存在一个文件中较为困难
+ 数据库
    - 通用系统文件夹（gdb）
    - 多用户关系数据库管理系统（Oracle，Microsoft SQL Server、PostgreSQL、IBM Informix或IBM Db2等等）
    - 需要配合使用数据库管理系统（DBMS），数据库管理系统又分为关系型数据库和非关系型数据库
        * 结构化数据：指由二维表结构来逻辑表达和实现的数据，严格遵循数据格式与长度规范，也成作为行数
        * SQL语句（Structured Query Language）：数据查询、数据定义、数据操纵、数据控制等

#### 6.1 空间数据库
+ 三个类型
    - RDBMS扩展SDE空间数据库
    - 面向对象数据库（OO-DBMS）
    - 通用数据库扩展空间模块：空间数据管理的专用模块，定义了操纵点、线、面、圆等控件对象的API函数；但是空间数据结构不能由用户定义，使用上受到一定的限制

#### 6.2 空间查询
+ 空间信息查询：按照一定要求对地理信息系统所描述的空间实体及其空间信息进行访问，从众多空间实体中挑选出用户要求的空间实体机器相应的属性；包括属性查空间、空间查属性以及空间关系查询三种基本类型

#### 6.3 空间拓扑
+ 拓扑属性：在拓扑变换下能够保持不变的几何属性
+ 拓扑关系：描述了两个对象之间的拓扑属性之间的关系
    - 拓扑关系的意义：利用拓扑关系可以确定地理实体间的相对空间位置，而无需利用坐标和距离，利用拓扑关系有利于空间要素的查询
    - 九交模型：Egenhofer等人构造的9交空间关系模型（9I Model）；讨论的实际上是边界、内部和外部的相交或非相交的情况。共有2的9次方512种情况。
    - 九交扩展模型：6的9次方10077696种情况

### 7 第七章 GIS基本空间分析
#### 7.0 概述
+ 空间分析：指以空间数学模型与地理原理为依托，通过对空间（位置、形态）
+ 传统地图的分析方法：目视分析法、地图量算分析、图解分析、数学分析

#### 7.1 空间量算
+ 几何量算：点（坐标），线（长度、曲率、方向），面（面积、周长等）
+ 质心量算
+ 形状量算：形状系数目标物周长比上2倍根号pi乘上面积的平方根，结果<1是紧凑型，>1是膨胀型，=1是标准圆。
+ 距离量算：非标准欧氏距离，欧氏距离，曼哈顿距离

#### 7.2 叠加分析
+ 视觉信息复合，叠加分析模型，多边形叠加算法（先几何求交再属性分配）
+ 线缓冲区的生成：角平分线法、凸角圆弧法，然后还有自相交（岛屿和重叠）的处理
+ 栅格分析：栅格图层叠加、局部运算、二值逻辑叠加、多栅格的局部运算

### 8 第八章 数字高程模型与地形分析
+ TIN（不规则三角网），Voronoi图，Delaunay三角形
+ 空间插值分类
    - 精确插值：全局多项式插值、反距离权重插值、径向基插值、局部多项式插值
    - 近似插值：多种克里金插值（线性、无偏、最小估计方差）
    - 其他：样条法、趋势面插值（多元回归分析得到平滑数学平面方程）
+ 插值验证：交叉验证，“实际”验证 -> 本质上差不多

### 9 第九章 空间数据表现及制图
+ 视觉变量：包括基本视觉变量和派生视觉变量
    - **<font style="color:#DF2A3F;">基本视觉变量：形状、尺寸、方向（3个几何量），颜色色相、颜色亮度、颜色饱和度（3个颜色量）</font>**
    - 派生视觉变量：图案纹理、图案方向、图案排列
+ 视觉变量的感受：类别感、等级或顺序感、数量感、方向感、动态感
+ 图形符号：点状符号（不依比例）、线状符号（半依比例）、面状符号（依比例）、体积符号
    - 符号的大小：比率符号与非比率符号；比率符号分为连续与分级比率符号；连续比例符号分为绝对比率符号和条件比率符号（有没有数学转化）
+ 地图注记四要素：字体（表示不同事物）、字形，表示不同事物）、字号（面向重要程度）、字色（面向属性）
+ 注记排列方式三要素：字隔、字向、字列（屈曲字列还是雁行子列，特例为水平字列和垂直字列）、
+ 专题信息表示方法：定点符号法，线状符号法，动线法，底质法和范围法，分区统计图表法，定位统计图表法，等值线法/等值区域法
+ 其他的事情：视觉重量、视觉中心、制图综合
