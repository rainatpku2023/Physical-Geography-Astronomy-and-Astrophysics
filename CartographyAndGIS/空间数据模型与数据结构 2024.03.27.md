### 4.0 基本概念
+ 数据：指某一目标定性、定量描述的原始资料，包括数字、文字、符号、图形、图像以及他们能转换成的数据等形式
+ 信息：数据和消息中包含的意义，不随载体物理设备形式的改变而改变
+ 地理信息：
    - 最显著标志：位置通过数据进行标识
    - 特征：区域性特征、多为结构特征（多主体的第三维结构特征）、动态变化特征（时序特征）
        * 标识空间位置的数据成为空间位置数据

### 4.1 地理空间数据特征
+ 空间数据的基本性质
    - 空间相关性：与空间位置相关，具有特定的坐标和空间参照系
    - 尺度：从简单到复杂，可以从抽象到具体
    - 动态性：空间动态变化，时态变化
+ 空间数据分类:
    - 空间特征（定位数据）：坐标（分点、线、面），拓扑关系（栅格还是矢量）
        * 空间特征又称为几何特征和定位特征
        * **空间特征是GIS区别于其他软件的根本特征**
        * 基于做标牌省的数据：定量的度量信息（面积周长质心等）定性的空间关系（拓扑关系方位关系）
    - 属性数据（非定位数据）：名称、数值等等
    - 时间特征
        * 现象或物体的变化周期有超短期的、短期的、中期的长期的等等

+ **专题属性：按照以下列举顺序，精确程度不断提高**
    - **定名量表** nominal scales（有道翻译为：名义尺度）
        * 只根据地物或者现象本身的固有本质和内在特征进行区分，回答“是什么”的问题
        * 只涉及到质量，不反映数量
    - **顺序量表** ordinal scales
        * 刻画地理事物的等级序列，只是一个相对等级
    - **间隔量表** interval scales
        * 增加了距离信息，必须要有标准单位
    - **比率量表** ratio scales
        * 以明确的比率数值描述对象的数值差异
        * 等比数列是常用的方法之一
        * 可以直接用尺子去量一量获取数据

### 4.2 空间数据模型
+ 数据模型的概念：对现实世界的地理认知，构建世界而非复制世界
    - 数据模型具有主观性，每个人的关注点不同做出来的成果也不同
    - 对现实的表达具有选择性，因此是不完全的
+ 空间数据模型的抽象层次
    - 现实世界-概念模型-逻辑模型-物理模型；对应着现实世界-概念世界-数字世界
+ 空间数据模型的类型
    - 场模型 Field Model：表示二维或三维空间中被看做是连续变化的现象
    - 要素模型 Feature Model：也称为对象模型，刻画离散对象
    - 网络模型 Network Model：描述离散的对象，着重表示元素的联系等
+ 场模型
    - 场函数：单值函数、一般为欧式平面、可用等值线表示
        * 离散化方式：离散点、等高线、空间栅格、多边形
    - 离散化转连续函数时必然有误差
    - 属性域：
        * 可以包含以下几种类型：定名、顺序、间隔和比率
        * 支持空值，可以填写“None”等字样，但是不能空着

+ 场模型的特征
    - 空间自相关：数值聚集程度的一种量度
        * 正空间自相关：聚集的倾向
        * 负空间自相关：相互排斥的倾向
        * 空间自相关为0：属性和位置无关

**<font style="color:#DF2A3F;">三个重要模型：场模型、要素模型、网络数据模型</font>**

+ **场模型：栅格数据模型 Rastor**
    - 连续覆盖，将连续空间离散化
    - 可以分为规则和不规则
+ **要素模型：基于要素的空间模型强调了单个地理现象**
    - 欧氏空间的三类地物要素：点对象、线对象和多边形对象
    - 对应着矢量数据结构

+ **网络数据模型：通过路径连接多个地理现象中的连通情况**
    - 基本特征：节点数据见没有明显的从属关系，一个节点和可以和其他多个节点创建关系
    - 基本组成部分：点的集合和边的集合
    - 一些名词：有向图、无向图、结点、链接、前驱、后继、标号、权重
+ 时空数据模型
    - 时态变化的三种类型：	
        * 属性随着时间的变化
        * 空间特性随时间的变化（空间位置、形状、分布、拓扑关系）
        * 空间和属性同时变化
    - 分析工具：时空棱柱
+ 三维数据模型
    - 三维GIS：将三维坐标（x，y，z）作为独立参数进行空间实体对象的建模

### 4.3 空间数据结构
+ 空间数据结构：是适用于计算机系统存储、管理和处理的地理数据的逻辑结构，是地理实体的空间排列方式和相互关系的抽象描述
    - 本质上是对空间数据的一众理解和解释
+ 栅格数据结构
    - 定义：将研究区域分为大小均匀且紧密相邻的网格阵列，每个网格作为一个像元或像素，由行、列定义，每个网格包含一个代码表示该像素的属性类型或量值，或者记录指向属性数据的指针
    - 特点：数据结构简单，定位隐含，容易实现，易于实现，易于与遥感结合，但是数据量大
    - 分辨率：指单个网格所代表的显示地面覆盖面积的尺寸。
    - 位置：由栅格行、列号定义
    - 栅格数据结构的问题：混合像元
    - 栅格数据的编码方式（压缩存储空间、加快读取速度）
        * 直接栅格编码：将栅格数据看作一个数据矩阵，逐行(或逐列)记录代码数据。
        * 链码：用曲线的起始点的坐标和边界点的方向代码来描述曲线或边界的方法
        * 游程长度编码：按行扫描，将相邻等值的像元合并，并记录代码的重复个数或变化的位置
        * 四叉树编码：将2^n×2^n像元组成的图像按四个象限进行递归分割，并判断属性是否单一。如果不单一继续划分下去。

+ 矢量数据结构：通过记录坐标的方式尽可能精确地记表示点、线、多边形等地理实体，坐标空间设为连续，允许任意位置、长度和面积的精确定义。	
    - 一般情况下，矢量数据结构的精度比栅格数据结构高得多，其精度仅受数字化设备的精度和数值记录字长的限制。矢量数据结构允许最复杂的数据以最小的数据冗余进行存储。
    - 面的存储：坐标序列法、树状索引编码法（包括点文件、线文件和多边形文件）

+ 空间关系：空间关系是指地理空间实体之间相互作用的关系
    - 拓扑空间关系：实体之间相邻、相通、包含、相交；
        * 拓扑属性：在拓扑变换下能够保持不变的几何属性
        * 描述了两个对象之间的拓扑属性的关系
        * 拓扑关系的意义：可以确定地理实体之间的相对空间位置，无需利用坐标和距离
        * 拓扑关系的类型举例：邻接关系、关联关系、包含关系

    - 顺序空间关系：实体在地理空间的排列顺序，例如前后上下左右、东南西北
    - 度量空间关系：实体之间距离远近的关系
